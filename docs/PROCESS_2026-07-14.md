## 卡住的**真实根因**(我今天才发现)

我本机 `db.csfbquytxxjguuiacagu.supabase.co` 这个域名**只有 IPv6 地址**(`2406:da1c:4c7:f800::b388`),**没有 IPv4**。

nslookup 结果:
```
db.csfbquytxxjguuiacagu.supabase.co
Address: 2406:da1c:4c7:f800::b388   ← IPv6 only
```

本机的网络栈不支持出向 IPv6(ping IPv6 失败),所以:
- `pg` 走 `getaddrinfo` 失败 → ENOTFOUND
- pooler 走 IPv4 解析 → 报 "tenant not found" (Supavisor 拒绝未授权区域)

**这是机器问题,不是您项目的问题。** 您浏览器能进 Supabase SQL Editor 是因为它走 443 端口的 HTTPS — 那是另一条路。

## 我为什么没直接搞定

我手动试过的路径 + 卡点(透明):

| 路径 | 卡点 |
|---|---|
| `npm install -g supabase` 然后 `supabase db push` | npm 包 wrapper 在 win32-x64 拉不到原生二进制 |
| 直接下 EXE | GitHub release 服务器连接超时 |
| 直接 `pg.connect` 到 db.xxx.supabase.co:5432 | 本机无 IPv6 出口 |
| 直接 `pg.connect` 到各 region pooler | Supavisor 报 tenant 不存在 |
| GitHub Contents API 创建 workflow | PAT 无 workflow scope(必须网页手动) |
| Git push .github/workflows/* | 同上 |

**任何路径都卡在 "需要本机或 PAT 多 1 个权限"**。这台机器的能力范围到此为止。
